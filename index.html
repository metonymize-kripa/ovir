Opportunistic Verifiable Inference Runtime

Map-reduce for neural inference.

OVIR turns repeated neural work into reusable chunks: split the input, run many small local encoders, merge compact summaries, reuse cached artifacts, and fall back to the full model only when confidence is low.

Workloads first

The first target is not open-ended generation. It is high-volume inference where the same documents, prompts, embeddings, adapters, and ranking candidates are touched repeatedly.

01 / Reranking

Query-document scoring

Score thousands of candidate passages or documents without recomputing every token interaction from scratch.

Pain
Long inputs and repeated corpora.
Lever
Chunk-level encodings, summaries, and fallback.
02 / RAG

Repeated document pipelines

Enterprise corpora are embedded, reranked, summarized, and reused across many similar questions.

Pain
Duplicate preprocessing and repeated chunk inference.
Lever
Hash-addressed chunk artifacts.
03 / LLM serving

Long-context prefix reuse

Repeated system prompts, RAG context, agent history, and document prefixes should become reusable execution artifacts.

Pain
Prefill cost, KV transfer, hot-prefix recomputation.
Lever
Content-addressed KV blocks.
04 / Recsys

Ads, feeds, and ranking

Ranking systems depend on huge embedding tables, sparse feature access, model sharding, and strict p99 latency.

Pain
Random memory access and tail utilization.
Lever
Feature locality and hot embedding reuse.

Concrete toy model: chunk, score, merge

Take a BERT-style query-document reranker. The dense baseline lets every token attend to every other token. OVIR replaces that first-pass scorer with a divide-and-conquer pipeline that can run across many workers and cache every intermediate.

Example request

Given one query and a 2,048-token document, decide whether the document is relevant. The system splits the document into 32 chunks of 64 tokens, prepends the query to each chunk, runs a tiny shared encoder on every chunk, and then merges the resulting summaries into one document-level score.

1 / Divide

Small local encoders

Each chunk sees the query plus a small text window. This is where the parallelism happens: 32 chunk jobs can run on 32 workers, and repeated chunks can be skipped entirely from cache.

document = 32 × 64-token chunks
unit = hash(query, chunk, model, config)
cache hit = no neural work
2 / Conquer

Local evidence packets

Each worker emits a local relevance score, a few summary vectors, and an uncertainty estimate. The summary is not a full hidden state; it is the evidence needed by the next stage.

chunk output = score + summaries + uncertainty
summary count = 2–8 vectors
artifact = reusable receipt
3 / Merge

Tree or butterfly reducer

The merger combines neighboring summaries, then sections, then the whole document. This is the intuitive FFT-like part: information crosses the document in logarithmic rounds instead of one dense all-token matrix.

32 chunks → 16 → 8 → 4 → 2 → 1
merge rounds = log₂(chunks)
global work ≈ chunks × log(chunks)
Normalize the requestsame tokenizer, same model snapshot, same precision, same query text, same document chunk boundaries.
Resolve cached chunksif the query-chunk pair has already been scored under the same config, reuse the stored score and summaries.
Run missing chunksindependent workers run the tiny encoder only for cache misses.
Merge summariesa reducer combines local evidence into a document score and uncertainty estimate.
Escalate risky casesambiguous, adversarial, or global-dependency-heavy examples go to the full BERT teacher or a heavier reranker.
Record the receiptpersist hashes, cache hits, worker IDs, logits drift, latency, and fallback decision.

Why this is believable

The claim is not that arbitrary intelligence decomposes cleanly. The narrower claim is that many production inference calls are local, repetitive, and tolerant of a cheap first-pass scorer backed by escalation.

Secret 01

Most evidence is local

For reranking, classification, extraction, and many RAG tasks, the decisive evidence often lives in a small span, not every pairwise token interaction.

Secret 02

Global context can be compressed

The reducer only needs enough cross-chunk information to decide whether local evidence agrees, conflicts, or requires escalation.

Secret 03

The full model teaches the shortcut

A frozen BERT teacher supplies labels, logits, hidden-state targets, and hard examples, so the small system learns what information must survive the split.

Secret 04

Cache changes the economics

Once a chunk or query-chunk pair is scored, future requests reuse the artifact. The best win is avoiding work, not making every individual kernel faster.

Secret 05

Fallback protects quality

The system does not need to replace BERT everywhere. It must cheaply handle easy/common cases and identify when dense attention is worth paying for.

Secret 06

Verification is targeted

Receipts make it possible to recompute samples, detect drift, measure cache correctness, and quarantine unreliable workers or model variants.

Quality testMeasure the cheap scorer against the BERT teacher on reranking, classification, and long-document QA candidates.
Throughput testCompare dense BERT reranking against chunked scoring plus cached summaries over repeated corpora.
Escalation testTune the uncertainty threshold until fallback captures the examples where chunking loses global information.
Economic testReport cost per accepted answer, not raw model FLOPs: cache hits, transfers avoided, latency, and teacher calls.

Concrete use cases

The runtime is intentionally narrow: it does not generate code or replace inference engines. It decides what can be reused, where work should run, when to escalate, and how results are verified.

Reranking

Cheap first-pass scorer

Score many candidates with chunked encoders, then send only uncertain or top-ranked candidates to a heavier reranker.

RAG

Reusable document work

Cache embeddings, local scores, summaries, and model-prefix states for repeated document pipelines.

KV cache

Prefix-aware LLM router

Route requests to workers already holding matching model, tokenizer, prompt-prefix, and KV-block state.

Ranking

Embedding-shard scheduler

Group ranking requests by sparse feature overlap, shard locality, model snapshot, and latency target.

Model variants

Adapter placement

Keep compatible LoRAs and fine-tunes close to shared base weights, warm replicas, and batchable traffic.

Audit

Recompute avoidance receipts

Emit hashes, worker metadata, cache-hit evidence, fallback rate, latency, cost, and execution lineage for each result.

Depth of the contemplated stack

OVIR sits above execution engines and below applications: a scheduler, content-addressed store, cache fabric, and verification layer that can integrate Ray, MAX, Mojo, ONNX, Elixir, Nx, and REE-style receipts.

Distributed substrate

Ray tasks and actors; BEAM/Elixir supervision; Kubernetes or VM swarms for placement and failure recovery.

Execution engines

MAX, Mojo kernels, ONNX Runtime, vLLM, SGLang, Nx, Ortex, or other node-local inference backends.

Content identity

Hash addressing for models, graph fragments, prompts, tensors, summaries, KV blocks, kernels, and outputs.

Cache fabric

CAS over local disks, object stores, GPU memory, CPU RAM, and network-transfer-aware KV movement.

Receipts

Reproducibility boundary plus telemetry for what ran, what was reused, what escalated, and what was avoided.
