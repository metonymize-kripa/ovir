Offline Verified Inference Retrieval Network

Pretrain your corpus once. Compositions run in milliseconds on small models.

OVIR shifts expensive reasoning from query-time inference to offline corpus pretraining. An outer optimization loop—driven by a synthetic evaluation suite—bakes domain expertise, graph connections, and tool configurations directly into a prepared corpus. At query time, a small, cheap inner-loop model (3–7B) simply composes these pre-trained facts rather than generating reasoning over raw text.

Runtime & pretraining

OVIR splits execution into an offline pretraining loop and an online composition loop. Pretraining compiles raw assets into a domain-curated semantic graph; the runtime executes co-optimized tools to serve fast, cheap queries.

The two-loop pattern

Outer loop

Outer loop optimization

An expensive LLM + DSPy offline compiler parses the corpus, extracts entities (GLiNER), builds precomputed graph trajectories (Neo4j), and defines concept hierarchies (COBWEB). It iteratively tunes tool configurations and retrieval parameters to maximize performance against the synthetic evaluation suite.

Forcing function

Synthetic evaluation

A representative dataset of queries, tasks, and expected reasoning paths is generated offline from the domain model. This dataset acts as the forcing function for both loops, allowing automated DSPy optimization and rigorous quality verification before deployment.

Inner loop

Inner loop runtime

A fast, inexpensive model (3–7B) serves live queries by calling the co-optimized toolset over the prepared corpus. Because reasoning is reduced to structural composition rather than fresh generation over raw text, token costs and response latency drop dramatically.

Corpus pretraining

Characterize & modelDefine the domain's entities, relationships, and target query patterns. This forms the schema for the optimization loop.
Generate synthetic evalsUse a frontier LLM to synthesize a diverse, weighted regression suite representing real-world questions and expected reasoning traces.
Decompose & annotateRun zero-shot entity extraction (GLiNER) and probabilistic clustering (COBWEB) to segment the corpus into highly connected, annotated chunks.
Construct knowledge graphMap co-occurrence and transitive links into low-latency graph stores (Neo4j/FalkorDB), precomputed for fast neighborhood lookups.
Optimize tools & indicesRun the DSPy outer loop to iteratively tune BM25 weights, Cypher queries, and tool signatures against the synthetic evaluation suite.
Pretraining cost is amortized: The heavy frontier-model compute is spent once. Every subsequent query leverages this pre-compiled domain intelligence.

Runtime loop

Extract intentThe inner-loop model extracts entities and parameters from the user query using optimized DSPy signatures.
Traverse concept clustersTraverses the COBWEB hierarchy to immediately prune the search space to highly relevant concept boundaries.
Execute graph & text toolsRuns scoped BM25 keyword matching and Neo4j graph neighborhood lookups using the co-optimized tool interfaces.
Assemble & evaluateComposes the retrieved pre-verified fragments. Evaluates confidence against the domain schema and checks for information conflicts.
Escalate when uncertainIf confidence falls below the threshold, the runtime escalates the query to a larger model, real-time web search, or human review.
Log trace & telemetryEmits a full retrieval trace detailing every tool call, traversed node, and latency metric to inform the next outer-loop pretraining run.

Build target

Milestone 1

Synthetic eval & outer loop

Build the pilot domain model, generate the synthetic evaluation dataset (>= 500 cases), and stand up the outer-loop pretraining optimizer.

Passing conditions:
Synthetic dataset reproduces query patterns, outer loop completes baseline optimization.
Milestone 2

Co-optimized runtime

Deploy the inner loop with a 3–7B model executing the co-optimized tools over the pretrained corpus. Target: < 500ms P95 query latency.

Passing conditions:
Query execution < 500ms, quality matches or exceeds raw RAG on the synthetic eval suite.
Milestone 3

Telemetry & refresh

Set up production telemetry logging retrieval traces and a telemetry-driven pretraining refresh loop to handle information drift.

Passing condition:
Telemetry flags low-confidence drift, automatic outer-loop re-run restores quality.

Technical spine

The implementation plan keeps the stack narrow: automated evaluation, structured graph pretraining, concept routing, and tool-model co-optimization.

DSPy Outer Loop

Compiler optimization

Expensive LLM + DSPy compiler offline optimization. Tunes prepared corpus, tool signatures, and prompt parameters against the synthetic eval suite.

GLiNER + COBWEB

Concept routing

Zero-shot entity tagging and hierarchical probabilistic concept clustering to segment chunk space into a fast-traversing search tree.

Neo4j / FalkorDB

Graph RAG

Traversal-based Graph RAG layer that connects relationships and structural transitions vector search alone misses.

Co-Optimized Tools

Shared signatures

Shared, optimized Cypher query and BM25 tool schemas used across both Loops to flow updates directly into the runtime.

DSPy Inner Loop

Runtime model

Cheap, fast inference-time runtime model (3–7B) tuned to orchestrate tool lookups rather than generate expensive fresh reasoning.
