Opportunistic Verifiable Inference Router

Stop spending frontier compute on tasks that do not need it.

OVIR routes AI work through the cheapest valid path first: prior work, formal programs, constrained small models, SMC-guided search, and finally large LLMs only when necessary.

Compute ladder

OVIR treats large-model inference as a scarce resource. Each request climbs the ladder only when cheaper layers cannot produce a validated result.

Reuse prior verified workSearch cached artifacts, successful executions, prior answers, and known-good transformations.
Try deterministic synthesisUse parsers, schemas, type systems, SQL engines, planners, and executable templates before generation.
Constrain small modelsGenerate only inside formal languages such as JSON, SQL, Python ASTs, PDDL, or domain grammars.
Run verifier-guided SMCUse cheap potentials early, expensive validators selectively, and resampling to focus compute on promising candidates.
Escalate to large LLMsCall frontier-scale models only when uncertainty or validation failure justifies the cost.
Emit evidenceAttach validator traces, avoided-call metrics, cost/latency telemetry, and reproducible execution receipts where available.

Reference implementation shape

The initial build should stay open, small, and measurable: a Python orchestrator, a cache, a validator registry, SMC-controlled generation, and a reproducibility layer for model calls.

01

Router

Classifies work by output type, validator availability, risk, and compute budget.

02

Validator registry

Plugs in linters, unit tests, SQL execution, JSON Schema, PDDL validation, RDKit, and task-specific checks.

03

SMC controller

Uses programmable potentials to guide generation and resample candidates before wasted completions accumulate.

04

Reuse layer

Prioritizes content-addressed artifacts, semantic cache hits, and previously validated work products.

05

Model adapters

Supports local open models first, then larger hosted models as explicit fallback paths.

06

Audit trail

Reports accepted route, rejected routes, validation evidence, avoided large-model calls, and measured cost.
